# Documentation Display Flow

## 1. Demo Docs Flow

**Frontend:**
- User visits a demo URL (e.g., `/p/twilio-demo`).
- `PublicProjectView.jsx` uses `useParams()` to get the `slug` (e.g., `twilio-demo`).
- Checks if `slug` exists in `demoProjects` (from `lib/demoData.js`).
- If found, loads the demo project data directly from the frontend (no backend call).
- Renders the correct template component (e.g., `TwilioTemplate`) with the demo project data.
- No API/network request is made for demo docs.

**Summary:**
- All demo docs are rendered 100% client-side using static data and template components.

---

## 2. User Docs Flow (User Projects)

**Frontend:**
- User visits a user project URL (e.g., `/p/my-cool-api`).
- `PublicProjectView.jsx` uses `useParams()` to get the `slug` (e.g., `my-cool-api`).
- Checks if `slug` exists in `demoProjects`. If not, proceeds to fetch from backend.
- Calls `projectApi.getBySlug(slug)` (from `lib/api.js`).
- This triggers a GET request to `/projects/slug/:slug` on the backend.

**Backend:**
- `/projects/slug/:slug` route in `server/routes/project.js` receives the request.
- Normalizes the slug (lowercase, strips invalid chars, collapses dashes).
- Looks up the project in MongoDB by sanitized slug.
- If found, returns project data (repoName, techstack, generatedDocs, template, etc.).
- If not found, returns 404.

**Frontend (continued):**
- Receives the project data from backend.
- Renders the correct template component (e.g., `TwilioTemplate`, `ModernTemplate`, etc.) with the project data.
- If not found, displays "Project not found."

**Summary:**
- User docs are fetched from the backend by slug, then rendered client-side using the correct template component.

---

## Key Differences
- **Demo docs:** 100% client-side, no backend call, static data.
- **User docs:** Fetched from backend by slug, dynamic data, rendered with template components.

---

## Diagram

Demo Docs:
User → `/p/twilio-demo` → [Frontend: demoProjects] → [Template Component]

User Docs:
User → `/p/my-cool-api` → [Frontend: API call] → `/projects/slug/my-cool-api` (Backend) → [MongoDB] → [Project Data] → [Template Component]
