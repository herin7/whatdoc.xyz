────────────────────────────────────────────────────────────────────────
  PIPELINE OVERVIEW
────────────────────────────────────────────────────────────────────────

  runGenerationPipeline(projectId, repoUrl, llmProvider, byokOptions) runs
  3 steps sequentially. Each step updates MongoDB Project.status and emits
  SSE events via a global EventEmitter (engineEmitter).

  The third parameter `llmProvider` defaults to 'gemini'. The fourth
  `byokOptions` carries { customKey, targetModel } from BYOK headers.

  Status flow:  idle → scanning → analyzing → generating → ready
                                                         ↘ failed

  Pipeline is kicked off (non-blocking) from:
    server/controllers/projectController.js → createProject()


────────────────────────────────────────────────────────────────────────
  STEP 1 — SCANNING / CLONING                              ✅ DONE
────────────────────────────────────────────────────────────────────────

  Status: 'scanning'
  SSE events emitted:
    status → { status: 'scanning', message: 'Cloning repository…' }
    log    → { step: 'cloning', message: 'Cloning repository...' }
    log    → { step: 'cloning', message: 'Repository cloned — N entries found' }

  Implementation:
    - Creates temp dir: server/tmp/<projectId>  (fs.mkdir recursive)
    - Shallow clones repo via simple-git: git.clone(url, path, ['--depth', '1'])
    - Counts entries with fs.readdir({ recursive: true })
    - Temp dir is ALWAYS cleaned up in finally block regardless of outcome

  Security:
    - Shallow clone minimises disk / bandwidth usage
    - finally block guarantees fs.rm(tempPath, { recursive: true, force: true })
    - server/tmp/ is in .gitignore


────────────────────────────────────────────────────────────────────────
  STEP 2 — UNIVERSAL CODE INGESTION                        ✅ DONE
────────────────────────────────────────────────────────────────────────

  Status: 'analyzing'
  SSE events emitted:
    status → { status: 'analyzing', message: 'Aggregating multi-language codebase…' }
    log    → { step: 'analyzing', message: 'Scanning directory and concatenating whitelisted files...' }
    log    → { step: 'analyzing', message: 'Aggregation complete — concatenated N files into context window.' }

  Core function: extractArchitectureUniversal(repoPath) → returns { rawCode, fileCount }

  How it works:
    1. Sets up `ignore` instance with .gitignore + hardcoded blocked dirs
    2. Recursively walks the repo directory tree
    3. Each file passes through three gates before being ingested:
       a) .gitignore filter (via `ignore` npm package)
       b) Fat-Trimmer Blacklist (see below)
       c) ALLOWED_EXTENSIONS / ALLOWED_EXACT_NAMES whitelist
    4. Accepted files are read as UTF-8, run through the Regex Guillotine
       (see below), then concatenated with `--- FILE: path ---` headers
    5. Hard character cap: 900,000 chars (~225k tokens)

  Supports 20+ languages: JS, JSX, TS, TSX, Python, Java, Kotlin, Scala,
  C, C++, Go, Rust, Ruby, PHP, C#, Swift, plus Dockerfile, Makefile,
  Markdown, JSON, YAML.

  ── Fat-Trimmer Blacklist ──
    Purpose: Block files/directories that waste tokens with zero
             documentation value.

    BLOCKED_FILENAMES:
      package-lock.json, yarn.lock, pnpm-lock.yaml, Cargo.lock,
      Gemfile.lock, composer.lock, poetry.lock, .DS_Store, Thumbs.db

    BLOCKED_DIRS (injected into ignore filter):
      dist, build, out, .next, .nuxt, .output, __pycache__,
      .pytest_cache, coverage, __tests__, __test__, __mocks__,
      vendor, .gradle, .idea, .vscode

    isBlockedPattern(filename) → boolean:
      *.min.js, *.min.css, *.bundle.js, *.chunk.js, *.map,
      *.test.{js,jsx,ts,tsx,py,java}, *.spec.{js,jsx,ts,tsx,py,java}

  ── Regex Guillotine (minifyFileContent) ──
    Purpose: Lightweight per-file regex compression that strips noise
             before the file enters the combined string.

    Passes (in order):
      1. Strip block comments    /* ... */  (non-greedy)
      2. Collapse 3+ consecutive //… comment lines → single note
      3. Truncate base64 / data-URI blobs  → [BASE64 DATA TRUNCATED]
      4. Truncate long string literals (>500 chars) → [LONG STRING TRUNCATED]
      5. Truncate long single-line arrays/objects (>600 chars) → [DATA TRUNCATED]
      6. Collapse 3+ consecutive blank lines → 1

  Output shape returned:
    {
      rawCode:   string,   // all files concatenated with --- FILE: --- headers
      fileCount: number    // total files ingested
    }


────────────────────────────────────────────────────────────────────────
  STEP 3 — LLM DOCUMENTATION GENERATION                    ✅ DONE
────────────────────────────────────────────────────────────────────────

  Status: 'generating'
  SSE events emitted:
    status → { status: 'generating', message: 'AI is writing the documentation…' }
    log    → { step: 'generating', message: 'AI is writing the documentation...' }
    log    → { step: 'generating', message: 'Using Google Gemini (gemini-2.0-flash)' }
    log    → { step: 'generating', message: 'Documentation generated — N characters' }

  Service file: server/services/llm.js
  SDK:          @google/generative-ai (Google Gemini)
  Default model: gemini-2.5-flash  (BYOK users may choose any model)
  Env var:      GEMINI_API_KEYS (comma-separated, round-robin via keyManager)

  Provider factory pattern (server/services/llm.js):
    - PROVIDERS map holds factory functions per provider name
    - getLLMProvider(name) → returns { name, label, generate(rawCode, opts) }
    - listProviders() → returns array of { key, label } for the frontend
    - Easy to add OpenAI / Anthropic later — implement a factory, register
      it in PROVIDERS, done. No changes needed in engine.js.

  BYOK (Bring Your Own Key):
    - generate() accepts { customKey, targetModel, isPro }
    - If customKey provided → uses it; otherwise → round-robin getNextApiKey()
    - Round-robin rotator: server/utils/keyManager.js

  Token Guillotine (llm.js):
    - Free-tier cap: 800,000 chars (~200k tokens)
    - Truncates at nearest --- FILE: --- boundary to avoid mid-file garbage
    - Appends premium upgrade notice
    - BYOK users and Pro users bypass the cap

  System prompt instructs the LLM to:
    1. Receive raw concatenated source code (not JSON AST)
    2. Deduce framework from imports, file types, code patterns
    3. Generate a comprehensive README.md
       (overview, tech stack, getting started, env vars, project structure,
        API endpoints table, database models table)
    4. Generate an API_REFERENCE.md
       (routes grouped by resource, method, path, description, request body,
        response format, auth notes)
    5. Separate the two documents with: <!-- SPLIT -->
    6. Never hallucinate files/routes not in the provided code
    7. Use tables, code blocks, proper heading hierarchy
    Temperature: 0.3 (deterministic), maxOutputTokens: 65536

  How it works:
    1. engine.js Step 3 calls getLLMProvider(llmProvider) to get provider
    2. Passes contextMap.rawCode directly (already minified by Regex Guillotine)
    3. Calls provider.generate(rawCode, byokOptions) → returns combined Markdown
    4. Saves the full Markdown string to Project.generatedDocs in MongoDB
    5. Emits final log with character count

  Project model additions (server/models/Project.js):
    - llmProvider: { type: String, enum: ['gemini', 'openai'], default: 'gemini' }
    - generatedDocs: String  (stores the raw Markdown output)

  Backend route (server/routes/project.js):
    - GET /projects/providers — returns listProviders() for the frontend

  Frontend (client/src/pages/ConfigureProject.jsx):
    - LLM picker with two cards: Gemini (active) and OpenAI (disabled "Coming soon")
    - llmProvider state sent with project.create() API call

  Frontend API (client/src/lib/api.js):
    - project.getProviders() — calls GET /projects/providers


────────────────────────────────────────────────────────────────────────
  CLEANUP (finally block)                                   ✅ DONE
────────────────────────────────────────────────────────────────────────

  SSE events emitted:
    log → { step: 'cleanup', message: 'Temporary files removed' }
    log → { step: 'cleanup', message: 'Warning: cleanup failed' }  (on error)

  Runs in finally {} — guaranteed even on pipeline crash.
  Uses fs.rm(tempPath, { recursive: true, force: true }).
  Cleanup errors are logged but don't mask the original error.


────────────────────────────────────────────────────────────────────────
  SSE TRANSPORT                                             ✅ DONE
────────────────────────────────────────────────────────────────────────

  Route: GET /projects/:projectId/stream  (protected, authmware)
  File:  server/routes/project.js

  How:
    - Sets SSE headers (text/event-stream, no-cache, keep-alive)
    - Flushes headers + sends ": connected\n\n" comment
    - Subscribes to engineEmitter for status:<projectId> and log:<projectId>
    - Writes events as: "event: status\ndata: {...}\n\n"
    - 15-second keep-alive ping (": ping\n\n")
    - req.on('close') removes listeners + clears interval + ends response

  Auth for EventSource:
    - EventSource API does not support custom headers
    - authmware.js updated to also accept ?token=<jwt> query param
    - Frontend passes localStorage token as query param on SSE URL

  FRONTEND REAL-TIME TERMINAL                               ✅ DONE
────────────────────────────────────────────────────────────────────────

  Component: client/src/components/GenerationTerminal.jsx
  Page:      client/src/pages/DeployProgress.jsx
  Route:     /deploy/:projectId  (ProtectedRoute)

  GenerationTerminal component:
    Props: projectId (string), slug (optional string)

    Connection:
      - Opens EventSource to /projects/:projectId/stream?token=<jwt>
      - Listens for 'status' and 'log' named events
      - Appends each event to a logs[] state array
      - Auto-closes connection on terminal states (ready / failed)
      - Cleanup: closes EventSource on component unmount

    UI — "hacker terminal" vibe:
      - Dark bg-black with green monospaced text (font-mono text-[13px])
      - Title bar with traffic-light status dot (pulse animation while running)
      - Initial fake prompt: $ whatdoc generate --project <id>…
      - Each log line: [HH:MM:SS] [STEP_TAG] message
      - Colour-coded by pipeline step:
          yellow-400   → CLONING
          cyan-400     → ANALYZING
          purple-400   → GENERATING
          zinc-500     → CLEANUP / SYS
          red-400      → ERR
          emerald-400  → READY
      - Blinking cursor block while pipeline is running
      - useRef auto-scroll to bottom on each new log

    Footer bar (appears on terminal state):
      - success: green "Documentation generated successfully" + "View Documentation" link → /p/:slug
      - failure: red "Pipeline failed" + "Try Again" link → /import

  DeployProgress page:
    - Wraps GenerationTerminal in a full-page layout with header/logo
    - Reads projectId from route params (/deploy/:projectId)
    - Reads slug from sessionStorage (set by ConfigureProject before navigating)

  Flow:
    ConfigureProject → deploy button → POST /projects →
    navigate(/deploy/:projectId) → DeployProgress renders GenerationTerminal →
    SSE stream shows real-time cloning → analyzing → generating → ready →
    user clicks "View Documentation" → /p/:slug
