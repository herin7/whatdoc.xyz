── 2. AUTH SYSTEM ──────────────────────────────────────────────────────

  Backend:
    Routes: server/routes/auth.js
      POST /auth/signup   — public
      POST /auth/signin   — public
      GET  /auth/me       — protected (authmware)

    Controller: server/controllers/authController.js
      signup()  — Validates with Zod (fname, lname, email, password min 6).
                  Creates user in MongoDB with bcrypt-hashed password.
                  Returns JWT token + user object (id, firstName, lastName, email).
      signin()  — Validates with Zod. Finds user by email, compares bcrypt hash.
                  Returns JWT token + user object.
      getMe()   — Reads req.userId from middleware, returns user (excludes
                  password & githubAccessToken).

    Middleware: server/middlewares/authmware.js
      - Reads Authorization: Bearer <token> header
      - Verifies JWT, sets req.userId
      - Returns 403 if invalid/missing

    User Model: server/models/User.js
      Fields: email (unique), password, firstName, lastName,
              githubId (sparse), githubUsername, githubAccessToken

    CORS: server/server.js
      - Origin set to process.env.CLIENT_URL (fallback http://localhost:5173)
      - credentials: true

  Frontend:
    API utility: client/src/lib/api.js
      apiRequest()     — Base fetch wrapper. Reads token from localStorage,
                         adds Bearer auth header. Throws on non-ok responses.
      auth.signup()    — POST /auth/signup
      auth.signin()    — POST /auth/signin
      auth.me()        — GET /auth/me

    AuthContext: client/src/context/AuthContext.jsx
      Provides: user, loading, login(), logout(), fetchUser()
      On mount: checks localStorage for token → calls /auth/me → sets user.
      login()  — stores token in localStorage + sets user state.
      logout() — removes token + clears user state.

    Routes: client/src/App.jsx
      /          → Home (public)
      /login     → Login (GuestRoute — redirects to /dashboard if logged in)
      /signup    → Signup (GuestRoute)
      /dashboard → Dashboard (ProtectedRoute)
      /import    → ImportRepo (ProtectedRoute)

    ProtectedRoute: client/src/components/ProtectedRoute.jsx
      - If loading: shows spinner
      - If no user: redirects to /login
      - Otherwise: renders children

    GuestRoute: defined inline in App.jsx
      - If logged in: redirects to /dashboard
      - Otherwise: renders children

    Login page: client/src/pages/Login.jsx
      - Cal.com-inspired dark UI
      - Email + password fields, eye icon to toggle password visibility
      - Error display below form
      - Calls auth.signin() → login(token, user) → navigate /dashboard
      - Links to /signup

    Signup page: client/src/pages/Signup.jsx
      - First name, last name, email, password
      - Eye toggle for password
      - Error display (maps Zod error messages)
      - Calls auth.signup() → auto-login → navigate /dashboard
      - Links to /login

    Logo component: client/src/components/Logo.jsx

── 3. GITHUB OAUTH INTEGRATION ────────────────────────────────────────

  Backend:
    Routes:
      GET /auth/github          — protected — returns GitHub OAuth URL as JSON
      GET /auth/github/callback — public — GitHub redirects here after auth
      GET /auth/github/repos    — protected — fetches all user repos from GitHub

    Controller:
      githubAuth()     — Reads ?includePrivate=true/false query param.
                         If true: scope = "user:email,repo" (includes private).
                         If false: scope = "user:email,public_repo" (public only).
                         Passes req.userId as state for the callback.
                         Returns JSON: { url: "https://github.com/login/oauth/authorize?..." }

      githubCallback() — Receives ?code and ?state from GitHub.
                         Exchanges code for access_token via GitHub API.
                         Fetches GitHub user profile (/user endpoint).
                         Saves githubId, githubUsername, githubAccessToken to User doc.
                         Redirects browser to CLIENT_URL/dashboard?github=success
                         (or ?github=error&reason=... on failure).

      getRepos()       — Reads user's githubAccessToken from DB.
                         Paginates through GitHub API /user/repos (100/page,
                         sorted by updated, affiliation=owner).
                         Returns clean list: { username, repos: [{
                           id, name, fullName, private, url, description,
                           language, updatedAt, defaultBranch, starCount
                         }] }

    Env vars needed:
      GITHUB_CLIENT_ID, GITHUB_CLIENT_SECRET, GITHUB_CALLBACK_URL, CLIENT_URL

  Frontend:
    API utility (client/src/lib/api.js):
      github.getAuthUrl(includePrivate) — GET /auth/github?includePrivate=true/false
      github.getRepos()                 — GET /auth/github/repos

    Dashboard (client/src/pages/Dashboard.jsx):
